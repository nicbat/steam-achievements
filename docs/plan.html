03

How it works

A genuine three-step sequence — each step blocks on the one before it.

1

Grab a Steam Web API key

The onboarding screen links straight to steamcommunity.com/dev/apikey, explains the domain field (any placeholder like localhost works), and reassures the user the key is only ever used by the local script.

UI state: “key not yet needed in the browser” — it’s for the collector, not the site.

2

Run the collector locally → steam_data.json

The site provides the exact command and the script (the Python collector we already have, plus an optional zero-install Node/Deno one-liner). It resolves the user, walks every owned game, and writes one JSON file with playtime, per-game completion, and every achievement + global rarity + description.

Runtime ~3–4 min for a 200-game library · ~2 MB output.

3

Upload the file into the viewer

Drag-and-drop parses the JSON entirely in-browser and caches it in localStorage, so refreshes are instant and nothing is transmitted anywhere.

Re-run step 2 anytime to refresh; the viewer diffs old vs new if a previous snapshot exists.

04

The one hard problem: CORS

This decides the whole architecture, so it’s worth being explicit.

Constraint

The Steam Web API sends no Access-Control-Allow-Origin header. A browser therefore cannot call api.steampowered.com directly from a web page — the request is blocked before it leaves. So “a script that runs in the web, locally, with no server” can’t be a browser fetch. Here’s how each option really shakes out:

ApproachServer costKey stays privateVerdict
Downloadable local script
user runs it, uploads JSON		 None Yes — key never leaves the machine Recommended
Public CORS proxy
e.g. corsproxy.io in front of Steam		 None No — key transits a third party Rejected
Serverless function
Netlify/Vercel edge relays the call		 Small / free tier Partial — key hits your function Adds a backend
Browser extension
host_permissions bypass CORS		 None Yes Heavy to build

The recommended path is exactly the shape you described: a local generator + an upload-and-view app. The site’s job is to make the local script effortless to run — a copy button, an optional npx/deno run one-liner so there’s no install, and a clear “paste your key here” field that only ever templates the local command.

05

The viewer — what you get

Scanned and operated, not read top-to-bottom: summary first, then drill-down.

Profile overview

  • Avatar, level, account age, friends, total playtime
  • Headline tiles: games, achievements, avg completion, perfect count
  • Steam-accurate average (per-game floor, then mean)

Completion optimizer

  • Best single unlocks in started games, ranked by +avg
  • Almost-perfect games (1–3 from 100%)
  • New games worth starting — flags the ones that’d lower your average
  • “What-if”: check achievements, watch the projected % move

Achievement explorer

  • One searchable table of all 8,967, with description on expand
  • Sort by rarity, unlock date, game, locked/unlocked
  • Filter chips: unlocked, locked, ultra-rare (<5%), recent

Rarity insights

  • Rarest achievements you own (flex wall)
  • Rarest you’re missing, and the easy ones you skipped
  • Global-rarity distribution per game

Library & playtime

  • All games sortable by hours, completion, last played
  • Playtime-vs-completion scatter — spot neglected near-perfects
  • Per-platform hours (Windows / Mac / Linux)

Timeline & charts

  • Unlocks over time from unlock_time
  • Completion-rate histogram across your library
  • All charts respect light/dark and reduced-motion

07

Data model

Already emitted by steam_collect.py — the viewer’s single source of truth.

"player": { persona_name, steam_level, xp, country, account_created, bans{} },
"totals": { owned_games, total_playtime_hours, total_achievements, ... },
"games": [
  {
    appid, name, playtime_forever_hours, last_played,
    has_achievements, achievements_unlocked, achievements_total,
    completion_rate_pct,
    achievements: [
      { api_name, name, description, unlocked,
        unlock_time, global_completion_rate_pct }
    ]
  }
]

Note for the average: Steam floors each game’s percentage before averaging, so the viewer must compute floor(100 × unlocked / total) per game — that’s the difference between showing 33% and the real 32%.

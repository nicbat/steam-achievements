The Worth-Starting Finder

Two sliders that answer one question: which unplayed games would actually raise my average if I started them? A game only helps if you can realistically clear a bigger slice of it than your current average — this finds exactly those. Plan, not built yet.

1The idea

Starting a game adds it to your average at whatever completion you reach. If you’ll only scrape 10% of it, it drags you down; if you can comfortably clear 60%, it lifts you up. The trick is estimating “what can I realistically reach?” before committing.

Proxy for “realistically reachable”: achievements a lot of other players already have. An achievement at 45% global is usually a natural play-through unlock; one at 0.3% is a grind. So we count how many of a game’s achievements sit above a rarity floor, and ask whether that’s a big enough share.

Slider N — rarity floor

“Only count achievements at least N% global.” Your definition of easy enough to actually get. Higher N = stricter.

Slider X — completion target

“Show games I could take to at least X%.” Defaults to your current average, since anything above that is a net lift.

2The algorithm

for each candidate game g (unstarted, has achievements):
    easy      = count(a in g.achievements where a.global_rate >= N)   // "reachable" unlocks
    reachable = 100 * easy / g.achievements_total                     // % you'd plausibly clear
    qualifies = reachable >= X

    // where it lands you: this game enters your average at floor(reachable)%
    lift  = reachable > yourDisplayedAverage        // green if it pulls you up
    impact = projected average gain if you unlock those `easy` achievements   // plan-aware, exact

Sort qualifying games by reachable % (default) or by projected impact. Each row can add the reachable set (just the ≥N achievements) to your plan in one click — the most honest “worth starting” move — with the existing Plan to 100% and + easiest still available.

5How it fits & the edges

Integration

  • Replaces the static top-15 “Games worth starting” list with this slider-driven one.
  • Keeps the check-all Plan to 100% and + easiest actions; adds Plan easy set (the ≥N subset).
  • Slider positions persist (localStorage), like the plan pin.
  • All impacts stay plan-aware (the fix from this session) so numbers agree with the rest of the app.

Edge cases

  • Achievements with no global rate don’t count as easy (unknown difficulty).
  • X below your average surfaces games that’d still drag you down — I’ll mark those amber rather than hide them.
  • Optional toggle to include started-but-low games, not just untouched ones.
  • Empty result set shows a “loosen the sliders” hint.

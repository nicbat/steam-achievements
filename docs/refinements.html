Refinements · Plan v2

Six fixes to the plan & impact UI

Everything you flagged, grouped. Three of these (1–3) share one root cause: the impact of a single achievement is only meaningful in a game you’re already completing. The rest are display upgrades. Before/after mockups below.

01

Impact never hides — no more “0 until you click”

Now

Locked achievements in unstarted games show ~0, then a real number appears only after you add one — because the first achievement in a fresh game barely moves (or lowers) your average, and we were masking that as “~0”.

Fix

Always show the true signed impact: green +, red − (starting a big game one-at-a-time really does dip your average), grey 0.00 only for 100+ achievement games where one unlock genuinely rounds to nothing. Plus a one-line legend.

02

“Games worth starting” plans the whole game, not one achievement

Now

Clicking a game here adds one achievement. That’s right for Counter-Strike 2 (its only achievement = instant 100%), but for a 48-achievement game one unlock is meaningless — hence the confusion.

Fix

The primary action becomes Plan to 100% (a “finish” move) and the number shown is the +avg at 100%. One-achievement games behave exactly as before. A secondary “+ just the easiest one” stays available, clearly labelled with its real (often negative) single-unlock impact.

05

Selecting an achievement shows it selected everywhere

Ensure

Add an achievement in the Library and it reads ✓ in the Achievements list, Rarity walls, and the drawer — one shared identity (appid:apiName). The Optimizer’s game suggestions will reflect “in plan” when their target achievement (or a finish) is planned, so nothing looks unselected when it isn’t.

This mostly works today for specific achievements; the fix reconciles the Optimizer’s game-level suggestions with per-achievement selections so the states never disagree.
06

Achievement icons

Catch

Icon URLs come from a different Steam endpoint (GetSchemaForGame) that the collector doesn’t call yet — so they aren’t in your steam_data.json.

Fix

Add one schema call per game to the collector (stores icon for unlocked + gray icon for locked), then re-run it once (~154 extra calls, a few minutes). Images load straight from Steam’s CDN in the browser — no CORS issue. Then icons appear in the list, Library, Rarity, and plan drawer.

Decision

This is the only change needing a data re-fetch. Want me to add icons now (re-run the collector), or ship 1–5 first and add icons after?
